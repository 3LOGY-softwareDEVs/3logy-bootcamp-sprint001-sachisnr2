## ✅ PEER REVIEW BEST PRACTICES

### DO:
✅ Start with something positive  
✅ Be specific with examples  
✅ Suggest improvements, don't just criticize  
✅ Ask questions when you don't understand  
✅ Be kind and constructive  
✅ Test the code thoroughly  

### DON'T:
❌ Say "this is bad" without explaining why  
❌ Rewrite their code for them  
❌ Just say "LGTM" without actually reviewing  
❌ Focus only on negatives  
❌ Be vague ("needs work" - what kind of work?)  
❌ Personal attacks or mean comments

## 📤 HOW TO SUBMIT REVIEW

### Option 1: GitHub Issue (Recommended)
1. Go to their repository on GitHub
2. Click "Issues" tab
3. Click "New Issue"
4. Title: "Code Review from [Your Name]"
5. Paste your completed review
6. Submit issue

### Option 2: Discord
1. Save as file: `review-for-[name].md`
2. Post in #code-review channel
3. Tag the project owner
